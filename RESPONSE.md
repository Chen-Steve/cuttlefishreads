# Response Style

When working in this codebase, follow these response rules:

- Be concise and direct.
- Do not include a final summary unless explicitly requested.
- Do not include a “next steps” section unless explicitly requested.
- Do not explain every change unless explicitly requested.
- Do not over-explain obvious implementation details.
- Do not add new dependencies unless necessary.
- If a dependency is necessary, briefly explain why.
- Ask clarifying questions only when the request cannot be safely completed without more information.
- If the request is clear enough, make a reasonable assumption and proceed.
